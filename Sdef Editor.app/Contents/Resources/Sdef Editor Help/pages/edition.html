Terminology Edition

In Sdef Editor, each document contains one Dictionary, which can contain one or more Suites. A Suite contains Types, Classes, Commands and Events. Classes contains Properties, Elements, Commands and Events. This object hierarchy is represented in a Tree.

To edit content of an object, you have to select it in the tree view, and an editor panel will display (in the right part) details about the selected item (Figure 1).


Figure 1: Right part represents the selected Suite.
  • You can delete an object by selecting it in the tree and pressing delete key (⌫).
  • You can reorganise Objects in the Tree with Drag & Drop.
  • Commands & Events are in fact the same kind of objects. So you can drag & drop a Command in Events Folder and an Event in Commands Folder.

Some elements supports Synonyms and Documentation. To edit those fieds, you can use the Inspector panel (Figure 2).

: Synonyms and Documentation Elements are defined in the Sdef specification, but they are not always properly handled by the "Scripting Definition Processor" (sdp) that transforms sdef files into Cocoa or aete terminology.


Figure 2: Inspector Panel.

Inspector panel allows to add comments on every Objects. Comments will be render as standards XML comments in sdef files.

To Open Inspector panel you can either double clic on an Object in the Tree View, or use Windows Menu (⌘I).
